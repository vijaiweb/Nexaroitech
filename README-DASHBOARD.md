# Internship Flow, Dashboards & Certificates — What Was Added

## New pages
- **roadmap.html** — public page listing every domain's Week 1 / Week 2 / Final Task, plus the 15/30/45/60‑day duration options.
- **student-dashboard.html** — student register/login, daily task list, task submission, progress bar, certificate + QR once issued.
- **admin.html** — admin login (demo password `admin123`, set in `js/nexaro-data.js`), review/approve/reject submissions, auto-issues certificate when the final task is approved.
- **internships.html** — added a 7-step "How It Works" flow section (Register → Select Domain → Daily Tasks → Submit → Admin Review → Final Project → Certificate) and expanded the domain list to all 12 roadmap domains.
- **verify.html** — now checks locally-issued certificates (this browser) and `data/certificates.json`, and auto-verifies via `verify.html?id=NEXARO-2026-001` (the link encoded in each certificate's QR code).

## ⚠️ Important: this is a front-end prototype, not a real backend
There's no server or shared database. Everything (student accounts, task submissions, certificates) is stored in **`localStorage` in the browser**. That means:
- A student who registers on their phone won't see their tasks on their laptop.
- Certificates an admin issues are only verifiable **on that same admin's browser** — unless you export/re-bundle them (see below).
- The admin password lives in client-side JS, so it is not real security — anyone can view it in the page source.

This is intentional: you didn't have a backend yet, and a working click-through demo was the fastest way to show the full flow (register → tasks → review → certificate → QR verification) end to end.

## Making certificates verifiable for everyone
Right now `data/certificates.json` is the "shared" certificate list that `verify.html` checks for anyone, on any device. To make an admin-issued certificate verifiable site-wide:
1. Open the browser console on `admin.html` after issuing a certificate.
2. Run `copy(localStorage.getItem('nexaro_certificates'))` to copy the JSON.
3. Merge the new certificate entry into `data/certificates.json` and re-upload that file to your host.

## Upgrading to a real backend later
When you're ready for a real system, the parts to replace are isolated in `js/nexaro-data.js`:
- Swap the `localStorage` read/writes for API calls (e.g. `fetch('/api/students')`).
- Replace the plaintext admin password with real authentication (server session or JWT).
- Move certificate issuance server-side so `certificates.json` is generated automatically instead of manually merged.
- The task roadmap (`NEXARO_DOMAINS`) and day-splitting logic (`nexaroGenerateTasks`) can be reused as-is — just call them from your backend instead of the browser.

## Roadmap data model
Each domain has `week1`, `week2`, and `final` themes (see `js/nexaro-data.js` → `NEXARO_DOMAINS`). Daily tasks are auto-generated by splitting the chosen duration (15/30/45/60 days) roughly 40% / 40% / 20% across those three phases — the exact 5-day-per-week breakdown wasn't specified, so this proportional split was used as a reasonable default. Adjust `nexaroSplitDuration()` if you want a different ratio.
